# PoseVenture Design Rules

This document defines the design philosophy and UI rules for PoseVenture.

All UI components, pages, and layouts should follow these guidelines to ensure a consistent product identity.

PoseVenture should feel like a **premium venture discovery platform**, not a generic AI tool or playful startup app.

---

# 1. Product Design Philosophy

PoseVenture is a **venture thinking tool**, not an entertainment AI.

The design should communicate:

- clarity
- seriousness
- strategic thinking
- calm confidence

The UI should feel closer to:

- a premium SaaS product
- a venture strategy platform
- a thoughtful AI decision tool

Avoid designs that feel like:

- AI toy products
- productivity apps
- colorful startup templates
- playful chat interfaces

---

# 2. Visual Tone

The visual tone should be:

- clean
- calm
- professional
- spacious
- minimal

Key principle:

**Less noise, more clarity.**

Whitespace is preferred over visual decoration.

---

# 3. Color System

The color palette should be restrained and professional.

### Background

Primary background:
- very light neutral
- off-white
- soft slate

Examples:

- `#F8FAFC`
- `#F9FAFB`

---

### Text

Primary text:
- deep slate or near black

Examples:

- `#0F172A`
- `#111827`

Secondary text:
- muted gray

Examples:

- `#64748B`
- `#6B7280`

---

### Accent Color

Accent color should be used sparingly.

Recommended tone:
- muted blue
- blue-violet
- calm technology tone

Examples:

- `#4F46E5`
- `#3B82F6`

Accent color should mainly appear in:

- CTA buttons
- links
- key highlights

Avoid heavy gradients or neon colors.

---

# 4. Typography

Typography should feel modern, readable, and professional.

Recommended fonts:

- Geist
- Inter

Avoid decorative or playful fonts.

---

### Heading Style

Headlines should be:

- bold
- clear
- high contrast

Example sizes:

Hero headline:
- text-4xl
- text-5xl

Section title:
- text-2xl
- text-3xl

---

### Body Text

Body text should prioritize readability.

Example:

- text-base
- text-lg

Line height should be comfortable.

---

# 5. Layout Philosophy

Layouts should prioritize clarity and breathing space.

Guidelines:

- large spacing between sections
- minimal visual clutter
- left-aligned content blocks
- clear hierarchy

Sections should feel **calm and structured**, not dense.

---

# 6. Section Spacing

Use generous spacing between sections.

Typical section spacing:

py-24
py-32


Avoid tightly packed layouts.

---

# 7. Card Design

Cards are the main UI pattern for PoseVenture.

Cards should be:

- simple
- structured
- easy to scan

Guidelines:

- rounded-xl or rounded-2xl
- subtle border
- minimal shadow

Example style:

rounded-2xl
border
border-gray-200
bg-white
shadow-sm


Avoid heavy shadows.

---

# 8. Button Design

Buttons should feel confident and premium.

Primary CTA:

- solid accent color
- rounded-xl
- medium padding

Example:

px-6 py-3
rounded-xl
bg-indigo-600
text-white
font-medium


Secondary buttons:

- outline style
- subtle border

---

# 9. Hero Section

The hero section is the most important visual area.

It should include:

- strong headline
- short explanatory subtitle
- primary CTA
- optional secondary CTA

Hero should communicate the core idea quickly:

PoseVenture helps users discover **one clear venture opportunity**.

Avoid overly complex visuals.

---

# 10. Landing Page Structure

The recommended landing structure is:

1. Hero
2. Problem
3. Before → After transformation
4. Product output explanation
5. Business analysis teaser
6. Final CTA

Each section should feel purposeful and focused.

---

# 11. Interaction Style

Interactions should be:

- smooth
- subtle
- fast

Avoid excessive animation.

Use small transitions when helpful.

---

# 12. AI Product Positioning

The interface should **not feel like a chatbot**.

PoseVenture is a **structured discovery tool**.

Therefore:

- avoid chat-style UI
- avoid long conversation layouts
- emphasize structured inputs and structured outputs

---

# 13. Discovery Form Design

The discovery page should feel like a focused thinking interface.

Input fields should be:

- spacious
- clearly labeled
- easy to read

Labels should guide thinking rather than just naming fields.

Example:

Segment  
Who are you thinking about?

Symptom  
What interesting signal or change have you noticed?

Problem  
What frustration or unmet need exists?

---

# 14. Result Page Design

The result page should highlight the structured opportunity.

Use cards for:

- target
- problem
- insight
- opportunity

The opportunity card should visually stand out.

---

# 15. Design Consistency Rule

Every page should maintain consistent:

- spacing
- typography
- card style
- button style

Avoid introducing new styles unless necessary.

Consistency is more important than visual experimentation.

---

# 16. Future Expansion

As PoseVenture grows, the design system should remain:

- minimal
- calm
- professional

The product should scale into:

- venture analysis tools
- insight libraries
- opportunity dashboards

The design should already feel capable of supporting those future features.
